# Code of Conduct

## Our Pledge

We want `sharp` to be a practical and respectful open source project. Everyone participating in the project is expected to communicate clearly, assume good intent, and focus feedback on the work.

## Expected Behavior

- Be respectful and constructive.
- Keep discussions relevant to the project.
- Explain technical tradeoffs with concrete reasoning.
- Accept maintainers' decisions once a direction is chosen.
- Help make the project easier for new contributors to understand.

## Unacceptable Behavior

- Harassment, insults, or personal attacks.
- Discriminatory language or behavior.
- Trolling, repeated off-topic comments, or deliberate disruption.
- Publishing private information without permission.
- Any behavior that makes collaboration unsafe or hostile.

## Enforcement

Maintainers may remove comments, close issues, reject pull requests, or block participants when behavior violates this code of conduct.

If you need to report a concern, open a private channel with the maintainers when available. If no private channel exists yet, avoid posting sensitive personal information publicly and use the repository's issue tracker only for non-sensitive project concerns.

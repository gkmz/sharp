# 行为规范

## 我们的承诺

我们希望 `sharp` 是一个务实、尊重他人的开源项目。所有参与者都应清晰沟通、默认善意，并把反馈集中在具体工作上。

## 期望行为

- 尊重他人，反馈应具有建设性。
- 讨论内容应与项目相关。
- 用具体理由解释技术取舍。
- 一旦维护者确定方向，应尊重项目决策。
- 帮助新贡献者更容易理解项目。

## 不可接受行为

- 骚扰、侮辱或人身攻击。
- 歧视性语言或行为。
- 恶意刷屏、反复偏题或故意破坏讨论。
- 未经允许公开他人隐私信息。
- 任何让协作环境变得不安全或敌意化的行为。

## 执行方式

如果行为违反本规范，维护者可以删除评论、关闭 issue、拒绝 PR 或限制相关参与者。

如果需要报告问题，请优先通过私有渠道联系维护者。如果暂时没有私有渠道，请避免公开敏感个人信息，并仅将 issue tracker 用于非敏感的项目问题。

# 安全策略

## 支持版本

`sharp` 当前处于 1.0 之前。除非明确维护发布分支，安全修复会直接应用到主分支。

## 报告漏洞

请不要在公开 issue 中直接披露疑似漏洞细节，应先让维护者有机会进行评估。

推荐流程：

1. 如果已有安全联系渠道，请优先私下联系维护者。
2. 提供简洁描述、受影响命令或工具、复现步骤和潜在影响。
3. 不要提交真实生产环境密钥、token 或敏感数据。

如果暂时没有私有安全联系方式，可以先创建一个不包含漏洞细节的公开 issue，请求维护者提供私有沟通渠道。

## 安全范围

通常属于范围内的问题：

- 命令执行或文件访问超出用户意图。
- 参数、输出、日志或剪贴板路径中不安全地处理密钥。
- 解析器在不可信输入下导致进程崩溃。
- 对 `sharp` 有实质影响的依赖漏洞。

通常不属于范围的问题：

- 工具按设计回显用户输入。
- 无效输入返回错误时导致的错误结果。
- 用户 shell 环境或 `sharp` 外部剪贴板提供方的漏洞。
